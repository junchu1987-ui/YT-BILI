## 产品简介

这是一个本地运行的自动化工具，可以：

- 📺 **自动监控 YouTube 频道**，当有新视频发布时自动下载
- 📥 **下载单个视频或整个播放列表**
- 🎬 **自动添加片头** + 转码为 H264（B站兼容格式）
- 🚀 **一键上传至 Bilibili**，附带标题、描述、封面
- 🌐 **Web 界面管理**，在浏览器中可视化操作，无需命令行

## 环境要求

| 依赖 | 最低版本 | 说明 |
|------|---------|------|
| Python | 3.10+ | https://python.org |
| ffmpeg | 任意 | 视频处理，需加入 PATH |
| bun.exe | v1.3+ | 解密 YouTube n-challenge 签名 |
| biliup | 任意 | B站上传工具（pip 自动安装）|
| yt-dlp | nightly | 视频下载（pip 自动安装）|

## 安装步骤

### Step 1 — 解压文件包

将压缩包解压到任意目录，例如 `D:\YT_BI_Anti\`

### Step 2 — 放置必要文件

在解压目录中放入以下文件（**手动获取**）：

| 文件 | 获取方式 |
|------|---------|
| `bun.exe` | [GitHub Releases](https://github.com/oven-sh/bun/releases) → `bun-windows-x64.zip` → 解压取出 `bun.exe` |
| `youtube_cookies.txt` | 在 Chrome 中登录 YouTube → 使用扩展 [Get cookies.txt LOCALLY](https://chrome.google.com/webstore/detail/get-cookiestxt-locally/cclelndahbckbenkjhflpdbgdldlbecc) → 导出保存为 `youtube_cookies.txt` |

### Step 3 — 运行安装脚本

双击运行 `setup.bat`，它会自动：
1. 检查 Python 环境
2. 创建虚拟环境 `.venv`
3. 安装所有 Python 依赖
4. 检查 ffmpeg / bun.exe

### Step 4 — 登录 B站

在终端中运行：
```bash
biliup login
```
选择"短信登录"或"扫码登录"，成功后会生成 `cookies.json` 文件。

## 首次配置

打开 `config.yaml` 或通过 **Web UI → 设置** 进行配置：

```yaml
app:
  proxy: "http://127.0.0.1:10808"   # 代理（不需要则留空）

bilibili:
  tid: 171    # 投稿分区 ID（171=汽车/摩托，17=单机游戏，等）
```

**常用 B站分区 ID**：

| ID | 分区 |
|----|------|
| 17 | 单机游戏 |
| 65 | 网络游戏 |
| 95 | 数码 |
| 171 | 汽车 |
| 193 | 电影 |

## 日常使用

### 启动 Web UI

双击 `run_web.bat`，然后在浏览器中打开：
```
http://127.0.0.1:5000
```

### 添加来源

1. 点击侧栏 **📋 来源管理**
2. 粘贴 URL → 点击 **+ 添加**

支持以下三种 URL 格式：

| 类型 | 示例 URL | 行为 |
|------|---------|------|
| **频道** | `https://www.youtube.com/@channelname` | 每次只下载最新 10 条中未处理的视频 |
| **单视频** | `https://www.youtube.com/watch?v=VIDEO_ID` | 下载该视频（已下过则跳过）|
| **播放列表** | `https://www.youtube.com/playlist?list=LIST_ID` | 下载全部未下载过的视频 |

### 运行流水线（分步操作）

点击侧栏 **🚀 流水线**，按照以下步骤操作：

**① 扫描** → 点击"🔍 开始扫描"  
系统会检测所有来源，列出发现的视频（标注哪些是新的）。

**② 下载** → 勾选要下载的视频 → 点击"⬇ 开始下载"  
实时显示每个视频的下载进度。

**③ 转码** → 确认后点击"🎬 开始转码"  
自动添加片头、统一编码为 H264。

**④ 上传** → 确认后点击"🚀 上传至 B站"  
上传完成后自动记录到历史，下次不会重复处理。

### 查看历史

点击侧栏 **🕐 历史记录**，可以看到所有已成功上传到 B站的视频 ID。

## 常见问题

### Q: 下载时显示"n challenge solving failed"
**A**: 确认 `bun.exe` 文件存在于程序根目录。检查方式：在根目录下运行：
```bash
.\bun.exe --version
```
应该输出版本号如 `1.3.11`。

### Q: 上传失败，提示 -400 错误
**A**: 可能是 B站 Cookie 过期。在终端运行：
```bash
biliup login
```
重新登录后即可。

### Q: 代理设置无效
**A**: 确认代理软件正在运行，且端口号正确。在"设置"页面更新代理地址并保存。

### Q: 视频无法在本地播放
**A**: 确认使用 VLC 或 Potplayer 播放 `_final.mp4` 文件。

### Q: 上传时一直无响应（挂起）
**A**: 可能是网络问题。检查代理设置，或等待超时重试。

## CLI 模式（高级）

如果不想使用 Web UI，也可以直接运行命令行：

```bash
# 运行完整流水线（扫描+下载+转码+上传，全自动）
python main.py

# 启动 Web UI
python web_app.py
```

日志文件：
- `pipeline.log` — CLI 模式日志
- `web_app.log` — Web UI 模式日志

## 云端部署（高级）

如需部署在 VPS 或云服务器上，对外网提供访问：

1. 编辑 `config.yaml`：
```yaml
app:
  host: "0.0.0.0"   # 监听所有网卡
  port: 5000
```

2. 建议在 Nginx 前置反向代理，并配置 HTTPS。

3. 生产环境使用 `gunicorn` 替代 Flask 开发服务器：
```bash
pip install gunicorn
gunicorn -w 1 -b 0.0.0.0:5000 web_app:app
```

> ⚠️ **安全警告**: 如果对外网开放，请务必在 Nginx 层添加 Basic Auth 或 IP 白名单，避免未授权访问。
