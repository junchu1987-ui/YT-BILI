## 🚀 快速开始（从 Git 下载）

如果您是第一次获取本工具，请按照以下步骤操作：

1. **克隆仓库**：
   打开终端（如 PowerShell 或 CMD），运行：
   ```bash
   git clone https://github.com/junchu1987-ui/YT-BILI.git
   cd YT-BILI
   ```

2. **准备二进制文件**：
   在项目根目录下创建一个 `bin` 文件夹，并将 `bun.exe` 放入其中。
   > [!TIP]
   > `bun.exe` 下载地址：[Bun Releases](https://github.com/oven-sh/bun/releases) (下载 windows-x64 版本)。

## 📦 安装与配置步骤

### 第一步：初始化环境
双击运行项目根目录下的 **`setup.bat`**。
该脚本非常强大，会自动完成：
- **自动检测并安装**：如果您的电脑缺少 `Git`、`FFmpeg` 或 `Python 3.11`，脚本会通过 Windows `winget` 尝试自动安装它们。
- **创建虚拟环境**：创建并配置 Python 虚拟环境 (`.venv`)。
- **安装依赖包**：自动安装项目所需的所有 Python 库。
- **检查环境**：确认一切就绪。

> [!TIP]
> 如果脚本安装了新工具（如 Git 或 FFmpeg），建议在脚本运行完后重启电脑或重新打开命令行窗口，以确保环境变量生效。

### 第二步：配置账号凭证 (Cookies)
为了确保能成功下载 YouTube 4K 视频并上传至 B站，您需要手动准备两个 Cookie 文件：

1. **YouTube Cookie**:
   - 在浏览器登录 YouTube。
   - 使用插件（如 "Get cookies.txt LOCALLY"）导出 Cookie。
   - 重命名为 `youtube_cookies.txt` 并存放在项目根目录。

2. **Bilibili Login**:
   - 在项目根目录打开终端，输入：
     ```bash
     .venv\Scripts\biliup login
     ```
   - 按照提示完成扫码登录。成功后根目录会生成 `cookies.json`。

### 第三步：修改基础设置
编辑 `config.yaml`（或稍后通过 Web 界面修改）：
- 设置您的 **网络代理** (如 `http://127.0.0.1:10808`)。
- 设置默认的 **B站投稿分区 ID**。

## 🖥️ 操作指南 (Web UI)

### 1. 启动程序
双击 **`run_web.bat`**。
程序启动后，会自动打开浏览器并访问：`http://127.0.0.1:5000`。

### 2. 视频来源管理
- 点击 **"📋 来源管理"**。
- 输入 YouTube 频道 URL、播放列表 URL 或单个视频 URL。
- 点击 **"添加"**。

### 3. 执行流水线
进入 **"🚀 流水线"** 页面，依次执行以下四个步骤：
- **① 扫描**：发现所有待处理视频。
- **② 下载**：将视频下至本地。
- **③ 转码**：自动调整分辨率并添加片头。
- **④ 上传**：发布至 B站。

## 💻 跨电脑迁移指南

如果您需要将工具移动到另一台电脑运行，请遵循以下“灵魂迁移”步骤，无需重新配置：

### 1. 备份关键文件（原电脑）
请将以下 5 个文件/文件夹拷贝出来：
- `config.yaml` (所有设置)
- `youtube_cookies.txt` (YouTube 登录)
- `cookies.json` (B站登录)
- `sources.json` (您的订阅来源列表)
- `history.json` (已处理视频的指纹，**非常重要，防止在新电脑重复上传旧视频**)

### 2. 在新电脑部署
1. 按照 [快速开始](#快速开始从-git-下载) 步骤克隆代码。
2. 运行 `setup.bat` 初始化环境。
3. 将上述备份的 5 个文件**覆盖**到新电脑的项目根目录。
4. 确保新电脑已安装 **FFmpeg**。

### 3. 验证
双击 `run_web.bat`。您会发现所有的来源、历史记录和登录状态都已经完美还原。

## ❓ 常见问题 (FAQ)

**Q: 为什么日志文件夹里是空的？**  
A: 系统现在采用集中日志管理。所有运行日志都存放在 `/logs` 目录下，您可以随时查看 `web_app.log` 或 `pipeline.log`。

**Q: `setup.bat` 提示找不到 `bin\bun.exe`？**  
A: 请确保您手动创建了 `bin` 文件夹，并把下载的 `bun.exe` 放进去了。不再支持直接放在根目录。

**Q: 代理连接超时？**  
A: 请确认您的代理软件开启了“允许局域网连接”或端口号与 `config.yaml` 中的配置一致。
